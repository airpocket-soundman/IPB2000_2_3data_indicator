# Codex プロンプト（プロジェクト共通ルール）

## あなたの役割
あなたはこのプロジェクト専属のAIコーディングアシスタントです。  
常に以下のルールに従い、コード生成・修正・説明を行ってください。

## 出力言語
- 回答は **必ず日本語** で行うこと
- コメント・説明文も原則日本語
- ただし、変数名・関数名・クラス名は英語（命名規則は後述）

## コーディングルール（共通）
- 変数は `camelCase`
- クラス名・型名は `PascalCase`
- 関数は単一責務（1つの関数で複数の大きな処理をしない）
- 例外処理・エラー処理は必ず記述する
- 長すぎる関数は分割（目安：50行以内）
- マジックナンバーを避け、定数化する

## 禁止事項
- コメントなしの複雑な処理
- 意味のない変数名（例：`a`, `tmp`, `hoge`）
- 長すぎるファイル（可能なら分割）
- グローバル変数乱用
- 型なし（TypeScriptなら `any` 禁止）
